ModelQA 1.4.0

オフラインで使える Windows 向け3Dモデル検証ツールです。FBX / glTF / GLBを開き、見た目や材質、UV、Animation、品質Issueをひとつの画面で確認できます。

まずはモデルを開いて眺めるだけでも大丈夫です。必要になったところから、マテリアル編集やレポート出力へ進めます。

  • Version 1.4.0
  • 形式 Windows 10 / 11 x64 Portable
  • 動作 オフライン

この配布はTechnicalUnsignedです。 技術検証に合格した未署名の候補であり、コード署名済みの公開版ではありません。Windowsの警告が出た場合は、入手元とファイルが正しいことを確認してから判断してください。

まずは3ステップ

1

フォルダーを展開

ZIPを受け取った場合は、先に書き込み可能なフォルダーへすべて展開します。ZIPの中から直接起動しないでください。

2

ModelQAを起動

ModelQA.exe をダブルクリックします。ModelQA.pak は同じ場所に置いたままにしてください。

3

モデルを開く

モデルをウィンドウへドロップするか、Ctrl+O で選びます。読み込みが終わったら、モデルを回して気になる箇所を確認しましょう。

気軽に試せます。 FBXを開いても原本は変更しません。モデルやマテリアルの編集結果を残すときはglTFまたはGLBとして書き出します。FBXのNode / Bone / Animation名の編集は、原本を変えずに .modelqa.names.json へ保存することもできます。

対応形式

読み込み

FBX / glTF / GLB に対応します。ファイル選択、ドラッグ＆ドロップ、「プログラムから開く」を利用できます。

書き出し

glTF / GLB に対応します。FBXへは書き戻さず、FBX原本は変更しません。

FBXの材質や構造は、glTFへそのまま対応できない場合があります。書き出したモデルは、いちどModelQAで開き直して見た目とIssueを確認してください。

WORKSPACES

画面とワークスペース

目的に合わせてパネルの並びを切り替えられます。モデルを開いた直後は「モデル確認」から始めるのがおすすめです。切り替えても、保存していないマテリアル編集はそのまま残ります。

ワークスペース向いている作業
モデル全体の形、Mesh、法線、Skin、基本情報をひと通り確認します。
マテリアルPBRテクスチャや係数を編集し、3D表示ですぐ確認します。
UVUV0 / UV1の配置、重なり、範囲外を確認します。
アニメーションAnimationクリップとタイムラインを確認します。
Issue QA検出されたIssueを重要度別に追い、該当箇所へ移動します。
カスタム自分で調整したウィンドウ配置を保存して使います。

組み込みワークスペースは Ctrl+1 ～ Ctrl+5、保存済みのカスタムは Ctrl+0 で切り替えられます。配置に迷ったら Ctrl+Shift+L で現在のレイアウトを初期位置へ戻せます。

MODEL REVIEW

モデルを確認する

3D表示

回転、移動、ズームを使い、表裏、輪郭、法線、反射を見ます。PBR材質はHDRIの反射も手がかりになります。

構造と品質

Mesh、Texture、Skin、Animationの情報とIssue一覧を確認します。Issueを選ぶと、対象を探しやすくなります。

読み込みに失敗した場合や取消した場合は、直前に確認していたモデルを保持します。読み込み中は保存操作が無効になるので、完了表示を待ってください。

MATERIALS

マテリアルを整える

「マテリアル」ワークスペースでは、次のPBR項目へ画像を選択またはドラッグ＆ドロップで割り当てられます。

  • Base Color — 表面の基本色と透明度
  • Normal — 細かな凹凸の向き
  • Metallic — 金属らしさ
  • Roughness — 反射のぼけ具合
  • AO — 隙間や接触部の陰影
  • Emissive — 自発光する色と強さ

Base Color、Metallic、Roughness、Normal強度、AO強度、Emissive色・強度を調整できます。各TextureはUV0 / UV1を選べます。

Metallic-Roughness自動パック: MetallicとRoughnessが別画像でも、glTF互換の1枚へ安全にまとめます。RoughnessはGチャンネル、MetallicはBチャンネルへ入ります。

編集は非破壊です。Undo / Redo で戻したり進めたりでき、プレビューは変更箇所だけを反映します。

UV / ANIMATION / QA

UV・Animation・Issue QA

UV

UV0 / UV1を切り替え、島の配置、重なり、0～1範囲の外側を確認します。大きいMeshでは確認用プレビューを使います。

Animation

クリップを選び、タイムラインを動かして姿勢の変化を確認します。SkinやAnimationのIssueもあわせて見てください。

Issue QA

重要度で絞り込み、内容と修正ヒントを確認します。必要なら Ctrl+R でレポートを出力し、制作側と共有できます。

SAVE / EXPORT / RECOVERY

保存・書き出し・復元

ModelQAには、名前編集を保存する通常のショートカットと、マテリアル編集を書き出す専用ボタンがあります。似ていますが、保存する内容とバックアップの扱いが違います。

Node / Bone / Animation名を保存

Ctrl+S / Ctrl+Shift+S はNode / Bone / Animation名の保存です。 Ctrl+S は現在のglTF / GLBへ名前を上書きします。通常のCtrl+Sではバックアップを作りません。

FBXで Ctrl+S を使った場合は、原本を変更せず .modelqa.names.json へ名前設定を保存します。Ctrl+Shift+S は名前を反映したモデルの別名保存で、FBXからの通常の別名保存はGLBだけです。

編集したマテリアルを書き出す

マテリアル画面の専用ボタン「GLB/glTFへ別名エクスポート...」は、Textureや係数を含む編集済みマテリアルをglTF / GLBへ書き出します。これは Ctrl+S / Ctrl+Shift+S とは別の操作です。

「元のglTF/GLBへ上書き」は、確認後に一時出力を検証してから元ファイルを置き換え、マテリアル上書きではバックアップを作ります。 FBXへは上書きできないため、専用の別名エクスポートを使ってください。

自動保存と復元: マテリアルの編集セッションは自動保存されます。異常終了後は復元候補を確認し、続きから作業できます。書き出したモデルはModelQAで開き直して見た目とIssueを確認すると安心です。

元モデルが外部で更新されている場合、自動保存した編集を黙って適用しません。表示される確認内容を読み、再対応付けまたは破棄を選んでください。

LARGE ASSET

巨大なFBXを開くとき

512MBを超えるFBX は、初期表示を待ち続けないよう、形状を最大600,000三角形へ縮約した Proxy Preview を使います。表示用法線も再生成します。

Proxy Previewは、全体形状や向きを早く確認するための軽量表示です。元データどおりの法線、Skin、Animation、Textureを完全に検査するものではありません。

完全な確認が必要なら、制作DCCで軽量化したFBXまたはGLBを書き出して開いてください。Proxy表示の結果を最終品質の証明には使わないでください。

DATA LOCATION

設定・キャッシュ・ログ・レポートの保存場所

アプリが管理する設定、キャッシュ、ログは、通常 %LOCALAPPDATA%\ModelQA をルートとして、その下の settings、cache、logs に保存します。そこへ書き込めない場合は %TEMP%\ModelQA\<現在のユーザーSID> をルートとして、同じサブフォルダーを使います。配布フォルダーへログやキャッシュを書き足しません。

ユーザーが出力したレポートは、既定で %USERPROFILE%\Documents\ModelQA\Reports に保存します。問い合わせに添える前に、モデル名やPC名など共有したくない情報がないか確認してください。

TROUBLESHOOTING

よくある問題

起動しない、またはModelQA.pakが見つからない

ZIPをすべて展開し、ModelQA.exe と ModelQA.pak が同じフォルダーにあるか確認してください。ファイルをひとつずつ別の場所へ移さないでください。

Windowsから未署名の警告が出る

このTechnicalUnsigned配布はコード署名されていません。入手元、配布者が案内したハッシュ、ファイル名を確認してください。確認できないファイルは実行しないでください。

モデルが開かない、読み込みが止まったように見える

巨大モデルは検査とProxy生成に時間がかかります。完了表示を待つか、Esc で取り消してください。繰り返し失敗する場合は、制作DCCから軽量化したFBXまたはGLBを出力して試します。

Textureが見つからない、見た目が違う

glTFと外部画像を一緒に移動したか、FBXとTextureの参照先が有効かを確認します。UV0 / UV1、Normal強度、Metallic-Roughnessの割り当ても見直してください。

画面が狭い、パネル配置が崩れた

目的に近いワークスペースへ切り替えるか、Ctrl+Shift+L で現在のレイアウトを初期位置へ戻してください。

保存できない

読み込み中でないか、出力先へ書き込めるかを確認してください。Node / Bone / Animation名を別名で残す場合、FBXの通常の別名保存はGLBです。 編集したマテリアルを残す場合は Ctrl+Shift+S ではなく、マテリアル画面の専用ボタンからglTF / GLBへエクスポートしてください。

PRIVACY / SAFETY

プライバシーと安全性

  • テレメトリなし: 利用状況やモデルを送信しません。
  • 自動更新なし: アプリが勝手に更新ファイルを取得しません。
  • 実行時ネットワークなし: モデル検証、ログ、レポート作成はローカルで完結します。
  • 隔離Import: 大きい入力や信頼できない入力は、制限付きの外部Import Workerで検査します。失敗や取消時は、直前の検証済みモデルを保持します。

オフラインでも、入手元の確認は必要です。 TechnicalUnsignedは未署名技術候補です。第三者から受け取った実行ファイルやモデルは、組織のルールに従って扱ってください。
